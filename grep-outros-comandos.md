O comando findstr é a ferramenta nativa do Prompt de Comando (CMD) do Windows para buscar padrões de texto em arquivos ou fluxos de dados, funcionando de forma semelhante ao grep do Linux.  Abaixo, detalho os comandos mencionados e as opções mais comuns para que você possa dominar esta ferramenta.Estrutura Básica do findstrO comando segue a sintaxe: findstr [opções] "string_de_busca" [arquivos].  Explicando os parâmetros citados/R: Utiliza a string de busca como uma expressão regular (Regular Expression), permitindo padrões mais complexos em vez de apenas texto simples.  /C:"string": Define o texto a ser buscado como uma string literal. Isso é essencial se você deseja buscar uma frase com espaços (ex: "log ativo"). Sem o /C, o findstr entenderia cada palavra como uma busca separada (buscando por "log" OU "ativo").  /B: Procura o padrão apenas no início de uma linha.  Exemplos Práticos Detalhados1. Filtrando uma lista de arquivosSe você quer filtrar a saída de um comando (como o dir) para encontrar apenas arquivos que contenham um termo específico, usamos o pipe |:  DOSdir /B | findstr /R /C:"[log]"
O que faz: Lista os nomes dos arquivos no diretório (dir /B) e passa essa lista para o findstr, que filtra apenas os nomes que contêm o termo "log".  2. Buscando dentro do conteúdo de arquivosSe você deseja procurar um termo dentro do texto de vários arquivos de uma vez:DOSfindstr "log" *
O que faz: Procura pela palavra "log" em todos os arquivos (*) do diretório atual. Ele exibirá o nome do arquivo seguido pela linha onde o termo foi encontrado.  Tabela de Opções ÚteisPara ampliar suas capacidades, aqui estão outras opções frequentemente utilizadas com o findstr:OpçãoDescrição/IIgnora maiúsculas e minúsculas (Case-insensitive)./SProcura arquivos no diretório atual e em todos os subdiretórios./NExibe o número da linha antes de cada linha que corresponde ao padrão./VImprime apenas as linhas que não contêm o padrão (inverte a lógica)./LUsa strings de busca literalmente (o oposto de /R).Dica Extra: Alternativa no PowerShellEmbora o findstr seja poderoso no CMD, no PowerShell o comando equivalente mais moderno e robusto é o Select-String (que pode ser abreviado para sls):  PowerShellGet-ChildItem | Select-String -Pattern "log"
O que faz: O Get-ChildItem lista os arquivos, e o Select-String varre o conteúdo de cada um procurando pelo padrão "log", oferecendo uma saída mais estruturada e fácil de manipular em scripts.
